# Native Asset Placeholders

Use this folder for source artwork before generating platform-specific assets.

- `athletixos-icon-1024.png` is the master 1024×1024 PNG used for iOS.
  It is regenerated from `public/brand/circle.PNG` via:
      sips -z 1024 1024 public/brand/circle.PNG \
        --out assets/native/athletixos-icon-1024.png -s format png
  and copied to `ios/App/App/Assets.xcassets/AppIcon.appiconset/AppIcon-512@2x.png`.
- The previous icon had heavy black padding around a small A mark; the
  current version uses the full brand circle so the logo fills the iOS
  rounded badge correctly.
- Android icons live in `android/app/src/main/res/mipmap-*/` and must be
  regenerated separately (e.g. via `npx @capacitor/assets generate
  --android` or Android Studio Image Asset Studio) when the source
  changes. Not auto-synced by `cap sync`.
- Keep club-specific logos in the web app and member portal branding
  settings, not in the native shell.

## Splash screen

- iOS splash lives in `ios/App/App/Assets.xcassets/Splash.imageset/` at
  `splash-2732x2732{,-1,-2}.png` (same source, three Contents.json slots).
- The splash badge was previously ~30% of the canvas with a visible
  black square around the brand circle. After iOS aspectFill cropped
  the square image onto a 9:19.5 phone, the badge rendered ~25% of
  screen height — too small.
- Current version is generated by the inline script committed alongside
  this README: a 2732x2732 charcoal (#1F1F23) canvas with the brand
  circle resized to 55% of canvas width and pasted through a circular
  alpha mask so the source's black square background blends seamlessly
  into the charcoal. Capacitor's `backgroundColor` already matches
  the charcoal so there's no flash transition into the WebView.
- To regenerate, re-run `python3 /tmp/gen_splash.py` from the splash
  script (kept inline in commit history) or write a similar Pillow
  script against `public/brand/circle.PNG`.
